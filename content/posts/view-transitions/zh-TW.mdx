過去要做跨頁轉場，往往需要框架、動畫函式庫，甚至刻意把頁面維持在同一個 SPA 生命週期中。View Transitions API 提供了一個更接近平台層的選項。

## 漸進增強比動畫更重要

最基本的原則是：沒有轉場時，導覽仍然要正常運作。動畫應該是體驗增強，而不是功能依賴。

```css
::view-transition-old(root) {
  animation: fade-out 160ms ease;
}

::view-transition-new(root) {
  animation: fade-in 220ms ease;
}
```

## 適合的使用情境

- 文章列表進入文章內容
- 圖片縮圖放大成詳情頁主視覺
- 同一資訊在不同版面間移動

## 不要忽略使用者偏好

動畫需要尊重 `prefers-reduced-motion`。轉場越華麗，越需要確認它真的協助理解介面狀態，而不是讓操作變慢。
