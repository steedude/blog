Server Components 剛被大量討論時，常被簡化成「把 React 搬到 Server」。幾年後回頭看，它真正改變的是元件邊界、資料存取位置，以及送到瀏覽器的 JavaScript 數量。

## 實現的期待

- Server 可以直接取得資料
- 不需要互動的元件不進入 client bundle
- UI 與資料需求可以維持在接近的位置

## 仍然存在的成本

最大的成本不是 API，而是心智模型。工程師需要知道哪些程式碼在哪裡執行、哪些資料能跨越邊界，以及快取策略如何影響使用者看到的內容。

> 一項架構能力是否成功，不只看它能不能工作，也要看團隊能不能穩定理解與維護。

## 現在的判斷

我不會因為專案使用 React 就自動選擇 Server Components。內容多、資料取得靠近 Server，而且希望控制 client JavaScript 時，它的價值最清楚。
