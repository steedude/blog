Core Web Vitals 提供共通語言，但分數本身不是使用者體驗。真正重要的是：使用者能否順利完成任務，以及問題發生在哪些裝置、網路與頁面流程。

## 實驗室資料與真實資料

本機測試適合重現問題，真實使用者監控則能看到分布。平均值容易掩蓋慢速裝置，因此我更在意百分位數與頁面類型。

```ts
type PerformanceEvent = {
  name: "LCP" | "INP" | "CLS"
  value: number
  route: string
  device: string
}
```

## 從指標回到操作

如果 INP 偏高，下一步不是直接「最佳化 React」，而是找出哪一個互動、哪一段主執行緒工作，以及使用者當時正在完成什麼任務。

## 我的檢查順序

1. 是否集中在特定頁面或裝置？
2. 是否影響核心操作？
3. 能不能穩定重現？
4. 改動後真實資料是否改善？
